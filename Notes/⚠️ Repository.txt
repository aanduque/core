#### External Types
Essa feature é, na verdade, uma evolução do bloco de query dinâmico, que era a minha abordagem até agora. A ideia é simples: ao invés de buscar uma estrutura de dados específica dentro do database do WordPress, nós utilizamos workflows para desempenhar as funções de retrieve clássicas de um CRUD.

#### Sistema de Suporte …
Isso é um mini-projeto, que quero utilizar de base para me ajudar a estruturar o trabalho de uma maneira menos abstrata. O primeiro passo é listar os requisitos dessa funcionalidade, na parte do front-end do usuário que vai acessar o painel do my-account. São elas:
- O usuário deve ser capaz de visualizar seus tickets em aberto e fechados dentro da página de suporte
- O usuário deve ser capaz de iniciar um novo ticket de suporte
	- Opcional: se possível, dar a opção de seleção da licença a ser atrelada ao ticket
- Uma vez criado, o usuário deve ser redirecionado a uma página dedicada a este ticket
- Se o ticket não existir, uma página de erro deve ser exibida, ou o usuário deve ser redirecionado para a listagem de tickets
- Dentro da página de detalhes de um ticket, o usuário deve ser capaz de ler a conversa em seu inteiro teor, e responder com uma nova mensagem
	- Optional: se for possível, o usuário deve ser capaz de enviar uploads em suas mensagens, bem como formatá-las com rich-text
- Optional: O usuário deve ser notificado quando uma nova resposta for recebida
- Optional: O usuário deve ser capaz de encerrar um ticket de suporte ao clicar em um botão dedicado a esta função
